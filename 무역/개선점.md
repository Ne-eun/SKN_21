# Baseline 개선 방안 분석

## 🎯 현재 Baseline의 한계점

### 1. **단순한 특성 공학 (Feature Engineering)**
- 현재: `b_t`, `b_t_1`, `a_t_lag`, `max_corr`, `best_lag`만 사용
- 한계: 시계열의 풍부한 정보를 활용하지 못함

### 2. **고정적인 임계값 설정**
- `corr_threshold=0.4`, `min_nonzero=12` 등이 하드코딩됨
- 품목별 특성을 고려하지 않음

### 3. **단일 모델 사용**
- LinearRegression만 사용
- 복잡한 비선형 관계를 포착하지 못함

### 4. **공행성 탐지 방법의 단순함**
- Pearson 상관계수만 사용
- 동적 상관관계나 조건부 관계 고려 안함

## 💡 개선 방안

## 🚀 구체적 개선 전략

### 1. **고급 특성 공학**
```python
# 시계열 통계 특성
- 이동평균 (3개월, 6개월, 12개월)
- 변동성 (rolling std, CV)
- 추세 (선형 회귀 기울기)
- 계절성 지수
- 증가율 (YoY, MoM)

# 품목별 특성
- 과거 N개월 평균/최대/최소값
- 품목별 거래 빈도
- 품목별 변동성 프로필
```

### 2. **다중 상관관계 분석**
- **Granger Causality Test**: 진정한 인과관계 검증
- **Dynamic Time Warping (DTW)**: 패턴 유사성 측정  
- **Mutual Information**: 비선형 의존성 탐지
- **Rolling Correlation**: 시간에 따른 상관관계 변화

### 3. **앙상블 모델링**
- **XGBoost/LightGBM**: 비선형 관계와 특성 상호작용 학습
- **LSTM/GRU**: 시계열 순차 패턴 학습
- **Transformer**: 어텐션 기반 장기 의존성
- **Stacking**: 여러 모델의 예측 결합

### 4. **데이터 전처리 고도화**
- **이상치 처리**: Isolation Forest, Z-score 필터링
- **결측값 보간**: 시계열 보간 (linear, spline, seasonal)
- **스케일링**: 품목별 정규화, Robust Scaling
- **데이터 증강**: 노이즈 추가, 시간 이동

### 5. **교차 검증 전략**
- **Time Series Split**: 미래 데이터 누수 방지
- **Purged Cross Validation**: 중복 정보 제거
- **Walk-Forward Analysis**: 실제 거래 환경 모사

### 6. **하이퍼파라미터 최적화**
- **Optuna/Hyperopt**: 자동 하이퍼파라미터 튜닝
- **베이지안 최적화**: 효율적인 탐색 공간 활용
- **다중 목적 최적화**: 정확도 vs 안정성 균형

## 🏆 공행성 쌍 추출을 위한 4단계 전략 요약

공행성 쌍을 찾는 대회에서 높은 점수를 받기 위한 신뢰성 있고 체계적인 4단계 분석 전략을 요약하여 제시합니다.

---

### 1. 🧪 데이터 준비 및 전처리 (Preparation & Preprocessing)

| 단계 | 목표 | 주요 함수 및 기법 |
| :--- | :--- | :--- |
| **정상성 확보** | 무역량 시계열의 추세(Trend)와 비정상성 제거. 허위 상관 방지. | **로그 차분(Log Differencing)**: $R_t = \ln(V_t / V_{t-1})$ |
| **이상치 처리** | 극단적인 값(이상치)의 영향 최소화. | **로버스트 스케일링** 또는 통계적 이상치 제거 기법 |
| **데이터 쌍 생성** | 100개 품목으로부터 가능한 모든 품목 쌍 생성. | **Combinations** (수학적 조합) |

---

### 2. 🛡️ 1차 공행성 쌍 추출 (선형 관계 기반)

| 단계 | 목표 | 주요 함수 및 기법 |
| :--- | :--- | :--- |
| **상관관계 계산** | 정상화된 데이터 간의 **선형적 공행성** 강도 측정. | **피어슨 상관계수** (`pd.Series.corr()`) |
| **필터링** | 높은 상관관계와 **통계적 유의성** 확보. | **상관계수 임계값** ($\mathbf{r > 0.45}$ 등) 적용. **P-value 임계값** ($\mathbf{P < 0.05}$ 또는 $\mathbf{P < 0.01}$) 적용. |

---

### 3. 🧩 2차 공행성 쌍 추출 (비선형 관계 및 보완)

| 단계 | 목표 | 주요 함수 및 기법 |
| :--- | :--- | :--- |
| **비선형 관계 탐색** | 선형 상관관계가 포착하지 못한 **비선형적 종속성**을 가진 쌍 보완. | **상호 정보량 (Mutual Information, MI)** 계산 |
| **쌍 추가** | 2단계에서 누락되었으나 상호 정보량이 높은 쌍을 잠정적인 공행성 쌍 목록에 추가. | MI 값이 **특정 임계값** 이상인 쌍을 추가. |

---

### 4. 🥇 최종 검증 및 선정 (신뢰성 및 안정성 확보)

2단계와 3단계에서 추출된 **모든 쌍**을 대상으로 다음 3가지 검증을 적용하여 최종 목록을 확정합니다.

| 검증 유형 | 목적 | 주요 함수 및 라이브러리 | 선정 기준 |
| :--- | :--- | :--- | :--- |
| **공적분 검증** | 쌍의 **장기적인 균형 관계** (경제적 유의성) 확인. (원 무역량 데이터 사용) | `coint(y1, y2)` (`statsmodels`) | $\mathbf{P-value} < \mathbf{0.1}$ 또는 $\mathbf{0.05}$ |
| **롤링 윈도우 분석** | 공행성 관계의 **시간적 안정성** 확인. | `pd.Series.rolling().corr()` (`pandas`) | **평균 상관계수**가 높고 **표준편차**가 낮아 안정적인 쌍 |
| **그랜저 인과성** | 쌍 간의 **선후행 관계 (방향성)** 확인. | `grangercausalitytests()` (`statsmodels`) | 유의미한 $\mathbf{P-value}$를 가진 쌍 ($\mathbf{P < 0.05}$) |
